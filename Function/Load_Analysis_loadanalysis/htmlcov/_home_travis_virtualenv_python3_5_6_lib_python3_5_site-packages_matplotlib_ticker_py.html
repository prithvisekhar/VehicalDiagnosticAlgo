2Tick locating and formatting 

3============================ 

4 

5This module contains classes to support completely configurable tick 

6locating and formatting. Although the locators know nothing about major 

7or minor ticks, they are used by the Axis class to support major and 

8minor tick locating and formatting. Generic tick locators and 

9formatters are provided, as well as domain specific custom ones. 

10 

11Default Formatter 

12----------------- 

13 

14The default formatter identifies when the x-data being plotted is a 

15small range on top of a large off set. To reduce the chances that the 

16ticklabels overlap the ticks are labeled as deltas from a fixed offset. 

17For example:: 

18 

19 ax.plot(np.arange(2000, 2010), range(10)) 

20 

21will have tick of 0-9 with an offset of +2e3. If this is not desired 

22turn off the use of the offset on the default formatter:: 

23 

24 ax.get_xaxis().get_major_formatter().set_useOffset(False) 

25 

26set the rcParam ``axes.formatter.useoffset=False`` to turn it off 

27globally, or set a different formatter. 

28 

29Tick locating 

30------------- 

31 

32The Locator class is the base class for all tick locators. The locators 

33handle autoscaling of the view limits based on the data limits, and the 

34choosing of tick locations. A useful semi-automatic tick locator is 

35`MultipleLocator`. It is initialized with a base, e.g., 10, and it picks 

36axis limits and ticks that are multiples of that base. 

37 

38The Locator subclasses defined here are 

39 

40:class:`AutoLocator` 

41 `MaxNLocator` with simple defaults. This is the default tick locator for 

42 most plotting. 

43 

44:class:`MaxNLocator` 

45 Finds up to a max number of intervals with ticks at nice locations. 

46 

47:class:`LinearLocator` 

48 Space ticks evenly from min to max. 

49 

50:class:`LogLocator` 

51 Space ticks logarithmically from min to max. 

52 

53:class:`MultipleLocator` 

54 Ticks and range are a multiple of base; either integer or float. 

55 

56:class:`FixedLocator` 

57 Tick locations are fixed. 

58 

59:class:`IndexLocator` 

60 Locator for index plots (e.g., where ``x = range(len(y))``). 

61 

62:class:`NullLocator` 

63 No ticks. 

64 

65:class:`SymmetricalLogLocator` 

66 Locator for use with with the symlog norm; works like `LogLocator` for the 

67 part outside of the threshold and adds 0 if inside the limits. 

68 

69:class:`LogitLocator` 

70 Locator for logit scaling. 

71 

72:class:`OldAutoLocator` 

73 Choose a `MultipleLocator` and dynamically reassign it for intelligent 

74 ticking during navigation. 

75 

76:class:`AutoMinorLocator` 

77 Locator for minor ticks when the axis is linear and the 

78 major ticks are uniformly spaced. Subdivides the major 

79 tick interval into a specified number of minor intervals, 

80 defaulting to 4 or 5 depending on the major interval. 

81 

82 

83There are a number of locators specialized for date locations - see 

84the `dates` module. 

85 

86You can define your own locator by deriving from Locator. You must 

87override the ``__call__`` method, which returns a sequence of locations, 

88and you will probably want to override the autoscale method to set the 

89view limits from the data limits. 

90 

91If you want to override the default locator, use one of the above or a custom 

92locator and pass it to the x or y axis instance. The relevant methods are:: 

93 

94 ax.xaxis.set_major_locator(xmajor_locator) 

95 ax.xaxis.set_minor_locator(xminor_locator) 

96 ax.yaxis.set_major_locator(ymajor_locator) 

97 ax.yaxis.set_minor_locator(yminor_locator) 

98 

99The default minor locator is `NullLocator`, i.e., no minor ticks on by default. 

100 

101Tick formatting 

102--------------- 

103 

104Tick formatting is controlled by classes derived from Formatter. The formatter 

105operates on a single tick value and returns a string to the axis. 

106 

107:class:`NullFormatter` 

108 No labels on the ticks. 

109 

110:class:`IndexFormatter` 

111 Set the strings from a list of labels. 

112 

113:class:`FixedFormatter` 

114 Set the strings manually for the labels. 

115 

116:class:`FuncFormatter` 

117 User defined function sets the labels. 

118 

119:class:`StrMethodFormatter` 

120 Use string `format` method. 

121 

122:class:`FormatStrFormatter` 

123 Use an old-style sprintf format string. 

124 

125:class:`ScalarFormatter` 

126 Default formatter for scalars: autopick the format string. 

127 

128:class:`LogFormatter` 

129 Formatter for log axes. 

130 

131:class:`LogFormatterExponent` 

132 Format values for log axis using ``exponent = log_base(value)``. 

133 

134:class:`LogFormatterMathtext` 

135 Format values for log axis using ``exponent = log_base(value)`` 

136 using Math text. 

137 

138:class:`LogFormatterSciNotation` 

139 Format values for log axis using scientific notation. 

140 

141:class:`LogitFormatter` 

142 Probability formatter. 

143 

144:class:`EngFormatter` 

145 Format labels in engineering notation 

146 

147:class:`PercentFormatter` 

148 Format labels as a percentage 

149

156 

157 ax.xaxis.set_major_formatter(xmajor_formatter) 

158 ax.xaxis.set_minor_formatter(xminor_formatter)

448 def pprint_val(self, x, d): 

449 """ 

450 Formats the value `x` based on the size of the axis range `d`. 

451 """ 

452 #if the number is not too big and it's an int, format it as an 

453 #int 

454 if abs(x) < 1e4 and x == int(x): 

455 return '%d' % x 

456 

457 if d < 1e-2: 

458 fmt = '%1.3e' 

459 elif d < 1e-1: 

460 fmt = '%1.3f' 

461 elif d > 1e5: 

462 fmt = '%1.1e' 

463 elif d > 10: 

464 fmt = '%1.1f' 

465 elif d > 1: 

466 fmt = '%1.2f' 

467 else: 

468 fmt = '%1.3f' 

469 s = fmt % x 

470 tup = s.split('e') 

471 if len(tup) == 2: 

472 mantissa = tup[0].rstrip('0').rstrip('.') 

473 sign = tup[1][0].replace('+', '') 

474 exponent = tup[1][1:].lstrip('0') 

475 s = '%se%s%s' % (mantissa, sign, exponent) 

476 else: 

477 s = s.rstrip('0').rstrip('.') 

478 return s

662 def _compute_offset(self): 

663 locs = self.locs 

664 if locs is None or not len(locs): 

665 self.offset = 0 

666 return 

667 # Restrict to visible ticks. 

668 vmin, vmax = sorted(self.axis.get_view_interval()) 

669 locs = np.asarray(locs) 

670 locs = locs[(vmin <= locs) & (locs <= vmax)] 

671 if not len(locs): 

672 self.offset = 0 

673 return 

674 lmin, lmax = locs.min(), locs.max() 

675 # Only use offset if there are at least two ticks and every tick has 

676 # the same sign. 

677 if lmin == lmax or lmin <= 0 <= lmax: 

678 self.offset = 0 

679 return 

680 # min, max comparing absolute values (we want division to round towards 

681 # zero so we work on absolute values). 

682 abs_min, abs_max = sorted([abs(float(lmin)), abs(float(lmax))]) 

683 sign = math.copysign(1, lmin) 

684 # What is the smallest power of ten such that abs_min and abs_max are 

685 # equal up to that precision? 

686 # Note: Internally using oom instead of 10 ** oom avoids some numerical 

687 # accuracy issues. 

688 oom_max = np.ceil(math.log10(abs_max)) 

689 oom = 1 + next(oom for oom in itertools.count(oom_max, -1) 

690 if abs_min // 10 ** oom != abs_max // 10 ** oom) 

691 if (abs_max - abs_min) / 10 ** oom <= 1e-2: 

692 # Handle the case of straddling a multiple of a large power of ten 

693 # (relative to the span). 

694 # What is the smallest power of ten such that abs_min and abs_max 

695 # are no more than 1 apart at that precision? 

696 oom = 1 + next(oom for oom in itertools.count(oom_max, -1) 

697 if abs_max // 10 ** oom - abs_min // 10 ** oom > 1) 

698 # Only use offset if it saves at least _offset_threshold digits. 

699 n = self._offset_threshold - 1 

700 self.offset = (sign * (abs_max // 10 ** oom) * 10 ** oom 

701 if abs_max // 10 ** oom >= 10**n 

702 else 0)

708 if not self._scientific: 

709 self.orderOfMagnitude = 0 

710 return 

711 if self._powerlimits[0] == self._powerlimits[1] != 0: 

712 # fixed scaling when lower power limit = upper <> 0. 

713 self.orderOfMagnitude = self._powerlimits[0] 

714 return 

715 locs = np.abs(self.locs) 

716 if self.offset: 

717 oom = math.floor(math.log10(range)) 

718 else: 

719 if locs[0] > locs[-1]: 

720 val = locs[0] 

721 else: 

722 val = locs[-1] 

723 if val == 0: 

724 oom = 0 

725 else: 

726 oom = math.floor(math.log10(val)) 

727 if oom <= self._powerlimits[0]: 

728 self.orderOfMagnitude = oom 

729 elif oom >= self._powerlimits[1]: 

730 self.orderOfMagnitude = oom 

731 else: 

732 self.orderOfMagnitude = 0 

733 

734 def _set_format(self, vmin, vmax): 

735 # set the format string to format all the ticklabels 

736 if len(self.locs) < 2: 

737 # Temporarily augment the locations with the axis end points. 

738 _locs = [*self.locs, vmin, vmax] 

739 else: 

740 _locs = self.locs 

741 locs = (np.asarray(_locs) - self.offset) / 10. ** self.orderOfMagnitude 

742 loc_range = np.ptp(locs) 

743 # Curvilinear coordinates can yield two identical points. 

744 if loc_range == 0: 

745 loc_range = np.max(np.abs(locs)) 

746 # Both points might be zero. 

747 if loc_range == 0: 

748 loc_range = 1 

749 if len(self.locs) < 2: 

750 # We needed the end points only for the loc_range calculation. 

751 locs = locs[:-2] 

752 loc_range_oom = int(math.floor(math.log10(loc_range))) 

753 # first estimate: 

754 sigfigs = max(0, 3 - loc_range_oom) 

755 # refined estimate: 

756 thresh = 1e-3 * 10 ** loc_range_oom 

757 while sigfigs >= 0: 

758 if np.abs(locs - np.round(locs, decimals=sigfigs)).max() < thresh: 

759 sigfigs -= 1 

760 else: 

761 break 

762 sigfigs += 1 

763 self.format = '%1.' + str(sigfigs) + 'f' 

764 if self._usetex: 

765 self.format = '$%s$' % self.format 

766 elif self._useMathText: 

767 self.format = '$%s$' % _mathdefault(self.format) 

768 

769 def pprint_val(self, x): 

770 xp = (x - self.offset) / (10. ** self.orderOfMagnitude) 

771 if np.abs(xp) < 1e-8: 

772 xp = 0 

773 if self._useLocale: 

774 return locale.format_string(self.format, (xp,)) 

775 else: 

776 return self.format % xp

778 def _formatSciNotation(self, s): 

779 # transform 1e+004 into 1e4, for example 

780 if self._useLocale: 

781 decimal_point = locale.localeconv()['decimal_point'] 

782 positive_sign = locale.localeconv()['positive_sign'] 

783 else: 

784 decimal_point = '.' 

785 positive_sign = '+' 

786 tup = s.split('e') 

787 try: 

788 significand = tup[0].rstrip('0').rstrip(decimal_point) 

789 sign = tup[1][0].replace(positive_sign, '') 

790 exponent = tup[1][1:].lstrip('0') 

791 if self._useMathText or self._usetex: 

792 if significand == '1' and exponent != '': 

793 # reformat 1x10^y as 10^y 

794 significand = '' 

795 if exponent: 

796 exponent = '10^{%s%s}' % (sign, exponent) 

797 if significand and exponent: 

798 return r'%s{\times}%s' % (significand, exponent) 

799 else: 

800 return r'%s%s' % (significand, exponent) 

801 else: 

802 s = ('%se%s%s' % (significand, sign, exponent)).rstrip('e') 

803 return s 

804 except IndexError: 

805 return s

823

826 the labeling of ticks that are not at integer powers of 

827 base; normally these are the minor ticks. The controlling 

828 parameter is the log of the axis data range. In the typical

1172 of 1000, plus a specified unit, e.g., 10 MHz instead of 1e7. 

1173 """ 

1174 

1175 # The SI engineering prefixes 

1176 ENG_PREFIXES = { 

1177 -24: "y", 

1178 -21: "z", 

1179 -18: "a", 

1180 -15: "f", 

1181 -12: "p", 

1182 -9: "n", 

1183 -6: "\N{GREEK SMALL LETTER MU}", 

1184 -3: "m", 

1185 0: "", 

1186 3: "k", 

1187 6: "M", 

1188 9: "G", 

1189 12: "T", 

1190 15: "P", 

1191 18: "E", 

1192 21: "Z", 

1193 24: "Y"

1196 def __init__(self, unit="", places=None, sep=" "): 

1197 """ 

1198 Parameters 

1199 ---------- 

1200 unit : str (default: "") 

1201 Unit symbol to use, suitable for use with single-letter 

1202 representations of powers of 1000. For example, 'Hz' or 'm'. 

1203 

1204 places : int (default: None) 

1205 Precision with which to display the number, specified in 

1206 digits after the decimal point (there will be between one 

1207 and three digits before the decimal point). If it is None, 

1208 the formatting falls back to the floating point format '%g', 

1209 which displays up to 6 *significant* digits, i.e. the equivalent 

1210 value for *places* varies between 0 and 5 (inclusive). 

1211 

1212 sep : str (default: " ") 

1213 Separator used between the value and the prefix/unit. For 

1214 example, one get '3.14 mV' if ``sep`` is " " (default) and 

1215 '3.14mV' if ``sep`` is "". Besides the default behavior, some 

1216 other useful options may be: 

1217 

1218 * ``sep=""`` to append directly the prefix/unit to the value; 

1219 * ``sep="\\N{THIN SPACE}"`` (``U+2009``); 

1220 * ``sep="\\N{NARROW NO-BREAK SPACE}"`` (``U+202F``); 

1221 * ``sep="\\N{NO-BREAK SPACE}"`` (``U+00A0``). 

1222 """ 

1223 self.unit = unit 

1224 self.places = places 

1225 self.sep = sep 

1226 

1227 def __call__(self, x, pos=None): 

1228 s = "%s%s" % (self.format_eng(x), self.unit) 

1229 # Remove the trailing separator when there is neither prefix nor unit 

1230 if self.sep and s.endswith(self.sep): 

1231 s = s[:-len(self.sep)] 

1232 return self.fix_minus(s)

1234 def format_eng(self, num): 

1235 """ 

1236 Formats a number in engineering notation, appending a letter 

1237 representing the power of 1000 of the original number. 

1238 Some examples: 

1239 

1240 >>> format_eng(0) # for self.places = 0 

1241 '0' 

1242 

1243 >>> format_eng(1000000) # for self.places = 1 

1244 '1.0 M' 

1245 

1246 >>> format_eng("-1e-6") # for self.places = 2 

1247 '-1.00 \N{GREEK SMALL LETTER MU}' 

1248 """ 

1249 sign = 1 

1250 fmt = "g" if self.places is None else ".{:d}f".format(self.places) 

1251 

1252 if num < 0: 

1253 sign = -1 

1254 num = -num 

1255 

1256 if num != 0: 

1257 pow10 = int(math.floor(math.log10(num) / 3) * 3) 

1258 else: 

1259 pow10 = 0 

1260 # Force num to zero, to avoid inconsistencies like 

1261 # format_eng(-0) = "0" and format_eng(0.0) = "0" 

1262 # but format_eng(-0.0) = "-0.0" 

1263 num = 0.0 

1264 

1265 pow10 = np.clip(pow10, min(self.ENG_PREFIXES), max(self.ENG_PREFIXES)) 

1266 

1267 mant = sign * num / (10.0 ** pow10) 

1268 # Taking care of the cases like 999.9..., which may be rounded to 1000 

1269 # instead of 1 k. Beware of the corner case of values that are beyond 

1270 # the range of SI prefixes (i.e. > 'Y'). 

1271 if float(format(mant, fmt)) >= 1000 and pow10 < max(self.ENG_PREFIXES): 

1272 mant /= 1000 

1273 pow10 += 3 

1274 

1275 prefix = self.ENG_PREFIXES[int(pow10)] 

1276 

1277 formatted = "{mant:{fmt}}{sep}{prefix}".format( 

1278 mant=mant, sep=self.sep, prefix=prefix, fmt=fmt) 

1279 

1280 return formatted

1335 +---------------+----------+------------------------+ 

1336 | >50 | 0 | ``x = 34.5`` => 35% | 

1337 +---------------+----------+------------------------+ 

1338 | >5 | 1 | ``x = 34.5`` => 34.5% |

1342 | ... | ... | ... | 

1343 +---------------+----------+------------------------+

1346 extremely large ones. It assumes that the values on the chart 

1347 are percentages displayed on a reasonable scale. 

1348 """

1350 if self.decimals is None: 

1351 # conversion works because display_range is a difference 

1352 scaled_range = self.convert_to_pct(display_range) 

1353 if scaled_range <= 0: 

1354 decimals = 0 

1355 else:

1362 decimals = 5 

1363 elif decimals < 0: 

1364 decimals = 0 

1365 else: 

1366 decimals = self.decimals 

1367 s = '{x:0.{decimals}f}'.format(x=x, decimals=int(decimals)) 

1368 

1369 return s + self.symbol

1923 def _raw_ticks(self, vmin, vmax): 

1924 """

1926 *vmax*. In some applications, one or both of the end locations 

1927 will not be needed, in which case they are trimmed off 

1928 elsewhere. 

1929 """ 

1930 if self._nbins == 'auto': 

1931 if self.axis is not None: 

1932 nbins = np.clip(self.axis.get_tick_space(), 

1933 max(1, self._min_n_ticks - 1), 9) 

1934 else: 

1935 nbins = 9 

1936 else: 

1937 nbins = self._nbins 

1938 

1939 scale, offset = scale_range(vmin, vmax, nbins) 

1940 _vmin = vmin - offset 

1941 _vmax = vmax - offset 

1942 raw_step = (_vmax - _vmin) / nbins 

1943 steps = self._extended_steps * scale 

1944 if self._integer: 

1945 # For steps > 1, keep only integer values. 

1946 igood = (steps < 1) | (np.abs(steps - np.round(steps)) < 0.001) 

1947 steps = steps[igood] 

1948 

1949 istep = np.nonzero(steps >= raw_step)[0][0] 

1950 

1951 # Classic round_numbers mode may require a larger step. 

1952 if rcParams['axes.autolimit_mode'] == 'round_numbers': 

1953 for istep in range(istep, len(steps)): 

1954 step = steps[istep] 

1955 best_vmin = (_vmin // step) * step 

1956 best_vmax = best_vmin + step * nbins 

1957 if best_vmax >= _vmax: 

1958 break 

1959 

1960 # This is an upper limit; move to smaller steps if necessary. 

1961 for istep in reversed(range(istep + 1)): 

1962 step = steps[istep]

1964 if (self._integer and 

1965 np.floor(_vmax) - np.ceil(_vmin) >= self._min_n_ticks - 1): 

1966 step = max(1, step) 

1967 best_vmin = (_vmin // step) * step 

1968 

1969 # Find tick locations spanning the vmin-vmax range, taking into 

1970 # account degradation of precision when there is a large offset. 

1971 # The edge ticks beyond vmin and/or vmax are needed for the 

1972 # "round_numbers" autolimit mode. 

1973 edge = _Edge_integer(step, offset) 

1974 low = edge.le(_vmin - best_vmin) 

1975 high = edge.ge(_vmax - best_vmin) 

1976 ticks = np.arange(low, high + 1) * step + best_vmin 

1977 # Count only the ticks that will be displayed. 

1978 nticks = ((ticks <= _vmax) & (ticks >= _vmin)).sum() 

1979 if nticks >= self._min_n_ticks: 

1980 break 

1981 return ticks + offset
