128 """ 

129 Convert a Hermite series to a polynomial. 

130 

131 Convert an array representing the coefficients of a Hermite series, 

132 ordered from lowest degree to highest, to an array of the coefficients 

133 of the equivalent polynomial (relative to the "standard" basis) ordered 

134 from lowest to highest degree. 

135 

136 Parameters 

137 ---------- 

138 c : array_like 

139 1-D array containing the Hermite series coefficients, ordered 

140 from lowest order term to highest. 

141 

142 Returns 

143 ------- 

144 pol : ndarray 

145 1-D array containing the coefficients of the equivalent polynomial 

146 (relative to the "standard" basis) ordered from lowest order term 

147 to highest. 

148

150 -------- 

151 poly2herm 

152 

153 Notes 

154 ----- 

155 The easy way to do conversions between polynomial basis sets 

156 is to use the convert method of a class instance. 

157

168 n = len(c)

238def hermfromroots(roots): 

239 """ 

240 Generate a Hermite series with given roots. 

241 

242 The function returns the coefficients of the polynomial 

243 

244 .. math:: p(x) = (x - r_0) * (x - r_1) * ... * (x - r_n), 

245 

246 in Hermite form, where the `r_n` are the roots specified in `roots`. 

247 If a zero has multiplicity n, then it must appear in `roots` n times. 

248 For instance, if 2 is a root of multiplicity three and 3 is a root of 

249 multiplicity 2, then `roots` looks something like [2, 2, 2, 3, 3]. The 

250 roots can appear in any order. 

251 

252 If the returned coefficients are `c`, then 

253 

254 .. math:: p(x) = c_0 + c_1 * H_1(x) + ... + c_n * H_n(x) 

255 

256 The coefficient of the last term is not generally 1 for monic 

257 polynomials in Hermite form. 

258 

259 Parameters 

260 ---------- 

261 roots : array_like 

262 Sequence containing the roots. 

263 

264 Returns 

265 ------- 

266 out : ndarray 

267 1-D array of coefficients. If all roots are real then `out` is a 

268 real array, if some of the roots are complex, then `out` is complex 

269 even if all the coefficients in the result are real (see Examples 

270 below). 

271 

272 See Also 

273 -------- 

274 polyfromroots, legfromroots, lagfromroots, chebfromroots, 

275 hermefromroots. 

276 

277 Examples 

278 -------- 

279 >>> from numpy.polynomial.hermite import hermfromroots, hermval 

280 >>> coef = hermfromroots((-1, 0, 1)) 

281 >>> hermval((-1, 0, 1), coef) 

282 array([ 0., 0., 0.]) 

283 >>> coef = hermfromroots((-1j, 1j)) 

284 >>> hermval((-1j, 1j), coef) 

285 array([ 0.+0.j, 0.+0.j]) 

286 

287 """ 

288 if len(roots) == 0: 

289 return np.ones(1) 

290 else: 

291 [roots] = pu.as_series([roots], trim=False) 

292 roots.sort() 

293 p = [hermline(-r, 1) for r in roots] 

294 n = len(p) 

295 while n > 1: 

296 m, r = divmod(n, 2) 

297 tmp = [hermmul(p[i], p[i+m]) for i in range(m)] 

298 if r: 

299 tmp[0] = hermmul(tmp[0], p[-1]) 

300 p = tmp 

301 n = m 

302 return p[0]

305def hermadd(c1, c2): 

306 """ 

307 Add one Hermite series to another. 

308 

309 Returns the sum of two Hermite series `c1` + `c2`. The arguments 

310 are sequences of coefficients ordered from lowest order term to 

311 highest, i.e., [1,2,3] represents the series ``P_0 + 2*P_1 + 3*P_2``. 

312 

313 Parameters 

314 ---------- 

315 c1, c2 : array_like 

316 1-D arrays of Hermite series coefficients ordered from low to 

317 high. 

318 

319 Returns 

320 ------- 

321 out : ndarray 

322 Array representing the Hermite series of their sum. 

323 

324 See Also 

325 -------- 

326 hermsub, hermmul, hermdiv, hermpow 

327 

328 Notes 

329 ----- 

330 Unlike multiplication, division, etc., the sum of two Hermite series 

331 is a Hermite series (without having to "reproject" the result onto 

332 the basis set) so addition, just like that of "standard" polynomials, 

333 is simply "component-wise." 

334 

335 Examples 

336 -------- 

337 >>> from numpy.polynomial.hermite import hermadd 

338 >>> hermadd([1, 2, 3], [1, 2, 3, 4]) 

339 array([ 2., 4., 6., 4.]) 

340 

341 """ 

342 # c1, c2 are trimmed copies 

343 [c1, c2] = pu.as_series([c1, c2]) 

344 if len(c1) > len(c2): 

345 c1[:c2.size] += c2 

346 ret = c1 

347 else: 

348 c2[:c1.size] += c1 

349 ret = c2 

350 return pu.trimseq(ret)

402def hermmulx(c): 

403 """Multiply a Hermite series by x. 

404 

405 Multiply the Hermite series `c` by x, where x is the independent 

406 variable. 

407 

408 

409 Parameters 

410 ---------- 

411 c : array_like 

412 1-D array of Hermite series coefficients ordered from low to 

413 high. 

414 

415 Returns 

416 ------- 

417 out : ndarray 

418 Array representing the result of the multiplication. 

419 

420 Notes 

421 ----- 

422 The multiplication uses the recursion relationship for Hermite 

423 polynomials in the form 

424 

425 .. math:: 

426 

427 xP_i(x) = (P_{i + 1}(x)/2 + i*P_{i - 1}(x)) 

428 

429 Examples 

430 -------- 

431 >>> from numpy.polynomial.hermite import hermmulx 

432 >>> hermmulx([1, 2, 3]) 

433 array([ 2. , 6.5, 1. , 1.5]) 

434 

435 """ 

436 # c is a trimmed copy 

437 [c] = pu.as_series([c]) 

438 # The zero series needs special treatment 

439 if len(c) == 1 and c[0] == 0: 

440 return c 

441 

442 prd = np.empty(len(c) + 1, dtype=c.dtype) 

443 prd[0] = c[0]*0 

444 prd[1] = c[0]/2 

445 for i in range(1, len(c)): 

446 prd[i + 1] = c[i]/2 

447 prd[i - 1] += c[i]*i 

448 return prd

451def hermmul(c1, c2): 

452 """ 

453 Multiply one Hermite series by another. 

454 

455 Returns the product of two Hermite series `c1` * `c2`. The arguments 

456 are sequences of coefficients, from lowest order "term" to highest, 

457 e.g., [1,2,3] represents the series ``P_0 + 2*P_1 + 3*P_2``. 

458 

459 Parameters 

460 ---------- 

461 c1, c2 : array_like 

462 1-D arrays of Hermite series coefficients ordered from low to 

463 high. 

464 

465 Returns 

466 ------- 

467 out : ndarray 

468 Of Hermite series coefficients representing their product. 

469 

470 See Also 

471 -------- 

472 hermadd, hermsub, hermdiv, hermpow 

473 

474 Notes 

475 ----- 

476 In general, the (polynomial) product of two C-series results in terms 

477 that are not in the Hermite polynomial basis set. Thus, to express 

478 the product as a Hermite series, it is necessary to "reproject" the 

479 product onto said basis set, which may produce "unintuitive" (but 

480 correct) results; see Examples section below. 

481 

482 Examples 

483 -------- 

484 >>> from numpy.polynomial.hermite import hermmul 

485 >>> hermmul([1, 2, 3], [0, 1, 2]) 

486 array([ 52., 29., 52., 7., 6.]) 

487 

488 """ 

489 # s1, s2 are trimmed copies 

490 [c1, c2] = pu.as_series([c1, c2]) 

491 

492 if len(c1) > len(c2): 

493 c = c2 

494 xs = c1 

495 else: 

496 c = c1 

497 xs = c2 

498 

499 if len(c) == 1: 

500 c0 = c[0]*xs 

501 c1 = 0 

502 elif len(c) == 2: 

503 c0 = c[0]*xs 

504 c1 = c[1]*xs 

505 else: 

506 nd = len(c) 

507 c0 = c[-2]*xs 

508 c1 = c[-1]*xs 

509 for i in range(3, len(c) + 1): 

510 tmp = c0 

511 nd = nd - 1 

512 c0 = hermsub(c[-i]*xs, c1*(2*(nd - 1))) 

513 c1 = hermadd(tmp, hermmulx(c1)*2) 

514 return hermadd(c0, hermmulx(c1)*2)

517def hermdiv(c1, c2): 

518 """ 

519 Divide one Hermite series by another. 

520 

521 Returns the quotient-with-remainder of two Hermite series 

522 `c1` / `c2`. The arguments are sequences of coefficients from lowest 

523 order "term" to highest, e.g., [1,2,3] represents the series 

524 ``P_0 + 2*P_1 + 3*P_2``. 

525 

526 Parameters 

527 ---------- 

528 c1, c2 : array_like 

529 1-D arrays of Hermite series coefficients ordered from low to 

530 high. 

531 

532 Returns 

533 ------- 

534 [quo, rem] : ndarrays 

535 Of Hermite series coefficients representing the quotient and 

536 remainder. 

537 

538 See Also 

539 -------- 

540 hermadd, hermsub, hermmul, hermpow 

541 

542 Notes 

543 ----- 

544 In general, the (polynomial) division of one Hermite series by another 

545 results in quotient and remainder terms that are not in the Hermite 

546 polynomial basis set. Thus, to express these results as a Hermite 

547 series, it is necessary to "reproject" the results onto the Hermite 

548 basis set, which may produce "unintuitive" (but correct) results; see 

549 Examples section below. 

550 

551 Examples 

552 -------- 

553 >>> from numpy.polynomial.hermite import hermdiv 

554 >>> hermdiv([ 52., 29., 52., 7., 6.], [0, 1, 2]) 

555 (array([ 1., 2., 3.]), array([ 0.])) 

556 >>> hermdiv([ 54., 31., 52., 7., 6.], [0, 1, 2]) 

557 (array([ 1., 2., 3.]), array([ 2., 2.])) 

558 >>> hermdiv([ 53., 30., 52., 7., 6.], [0, 1, 2]) 

559 (array([ 1., 2., 3.]), array([ 1., 1.])) 

560 

561 """ 

562 # c1, c2 are trimmed copies 

563 [c1, c2] = pu.as_series([c1, c2]) 

564 if c2[-1] == 0: 

565 raise ZeroDivisionError() 

566 

567 lc1 = len(c1) 

568 lc2 = len(c2) 

569 if lc1 < lc2: 

570 return c1[:1]*0, c1 

571 elif lc2 == 1: 

572 return c1/c2[-1], c1[:1]*0 

573 else: 

574 quo = np.empty(lc1 - lc2 + 1, dtype=c1.dtype) 

575 rem = c1 

576 for i in range(lc1 - lc2, - 1, -1): 

577 p = hermmul([0]*i + [1], c2) 

578 q = rem[-1]/p[-1] 

579 rem = rem[:-1] - q*p[:-1] 

580 quo[i] = q 

581 return quo, pu.trimseq(rem)

585 """Raise a Hermite series to a power. 

586 

587 Returns the Hermite series `c` raised to the power `pow`. The 

588 argument `c` is a sequence of coefficients ordered from low to high. 

589 i.e., [1,2,3] is the series ``P_0 + 2*P_1 + 3*P_2.`` 

590 

591 Parameters 

592 ---------- 

593 c : array_like 

594 1-D array of Hermite series coefficients ordered from low to 

595 high. 

596 pow : integer 

597 Power to which the series will be raised 

598 maxpower : integer, optional 

599 Maximum power allowed. This is mainly to limit growth of the series 

600 to unmanageable size. Default is 16 

601 

602 Returns 

603 ------- 

604 coef : ndarray 

605 Hermite series of power. 

606 

607 See Also 

608 -------- 

609 hermadd, hermsub, hermmul, hermdiv 

610 

611 Examples 

612 --------

614 >>> hermpow([1, 2, 3], 2) 

615 array([ 81., 52., 82., 12., 9.])

619 [c] = pu.as_series([c])

700 if cnt < 0: 

701 raise ValueError("The order of derivation must be non-negative") 

702 if iaxis != axis: 

703 raise ValueError("The axis must be integer") 

704 iaxis = normalize_axis_index(iaxis, c.ndim) 

705 

706 if cnt == 0: 

707 return c 

708 

709 c = np.moveaxis(c, iaxis, 0) 

710 n = len(c) 

711 if cnt >= n: 

712 c = c[:1]*0 

713 else: 

714 for i in range(cnt): 

715 n = n - 1 

716 c *= scl 

717 der = np.empty((n,) + c.shape[1:], dtype=c.dtype) 

718 for j in range(n, 0, -1): 

719 der[j - 1] = (2*j)*c[j] 

720 c = der 

721 c = np.moveaxis(c, 0, iaxis) 

722 return c

725def hermint(c, m=1, k=[], lbnd=0, scl=1, axis=0): 

726 """ 

727 Integrate a Hermite series. 

728 

729 Returns the Hermite series coefficients `c` integrated `m` times from 

730 `lbnd` along `axis`. At each iteration the resulting series is 

731 **multiplied** by `scl` and an integration constant, `k`, is added. 

732 The scaling factor is for use in a linear change of variable. ("Buyer 

733 beware": note that, depending on what one is doing, one may want `scl` 

734 to be the reciprocal of what one might expect; for more information, 

735 see the Notes section below.) The argument `c` is an array of 

736 coefficients from low to high degree along each axis, e.g., [1,2,3] 

737 represents the series ``H_0 + 2*H_1 + 3*H_2`` while [[1,2],[1,2]] 

738 represents ``1*H_0(x)*H_0(y) + 1*H_1(x)*H_0(y) + 2*H_0(x)*H_1(y) + 

739 2*H_1(x)*H_1(y)`` if axis=0 is ``x`` and axis=1 is ``y``. 

740 

741 Parameters 

742 ---------- 

743 c : array_like 

744 Array of Hermite series coefficients. If c is multidimensional the 

745 different axis correspond to different variables with the degree in 

746 each axis given by the corresponding index. 

747 m : int, optional 

748 Order of integration, must be positive. (Default: 1) 

749 k : {[], list, scalar}, optional 

750 Integration constant(s). The value of the first integral at 

751 ``lbnd`` is the first value in the list, the value of the second 

752 integral at ``lbnd`` is the second value, etc. If ``k == []`` (the 

753 default), all constants are set to zero. If ``m == 1``, a single 

754 scalar can be given instead of a list. 

755 lbnd : scalar, optional 

756 The lower bound of the integral. (Default: 0) 

757 scl : scalar, optional 

758 Following each integration the result is *multiplied* by `scl` 

759 before the integration constant is added. (Default: 1) 

760 axis : int, optional 

761 Axis over which the integral is taken. (Default: 0). 

762 

763 .. versionadded:: 1.7.0 

764 

765 Returns 

766 ------- 

767 S : ndarray 

768 Hermite series coefficients of the integral. 

769 

770 Raises 

771 ------ 

772 ValueError 

773 If ``m < 0``, ``len(k) > m``, ``np.ndim(lbnd) != 0``, or 

774 ``np.ndim(scl) != 0``. 

775 

776 See Also 

777 -------- 

778 hermder 

779 

780 Notes 

781 ----- 

782 Note that the result of each integration is *multiplied* by `scl`. 

783 Why is this important to note? Say one is making a linear change of 

784 variable :math:`u = ax + b` in an integral relative to `x`. Then 

785 :math:`dx = du/a`, so one will need to set `scl` equal to 

786 :math:`1/a` - perhaps not what one would have first thought. 

787 

788 Also note that, in general, the result of integrating a C-series needs 

789 to be "reprojected" onto the C-series basis set. Thus, typically, 

790 the result of this function is "unintuitive," albeit correct; see 

791 Examples section below. 

792 

793 Examples 

794 -------- 

795 >>> from numpy.polynomial.hermite import hermint 

796 >>> hermint([1,2,3]) # integrate once, value 0 at 0. 

797 array([ 1. , 0.5, 0.5, 0.5]) 

798 >>> hermint([1,2,3], m=2) # integrate twice, value & deriv 0 at 0 

799 array([-0.5 , 0.5 , 0.125 , 0.08333333, 0.0625 ]) 

800 >>> hermint([1,2,3], k=1) # integrate once, value 1 at 0. 

801 array([ 2. , 0.5, 0.5, 0.5]) 

802 >>> hermint([1,2,3], lbnd=-1) # integrate once, value 0 at -1 

803 array([-2. , 0.5, 0.5, 0.5]) 

804 >>> hermint([1,2,3], m=2, k=[1,2], lbnd=-1) 

805 array([ 1.66666667, -0.5 , 0.125 , 0.08333333, 0.0625 ]) 

806 

807 """ 

808 c = np.array(c, ndmin=1, copy=1) 

809 if c.dtype.char in '?bBhHiIlLqQpP': 

810 c = c.astype(np.double) 

811 if not np.iterable(k): 

812 k = [k] 

813 cnt, iaxis = [int(t) for t in [m, axis]] 

814 

815 if cnt != m: 

816 raise ValueError("The order of integration must be integer") 

817 if cnt < 0: 

818 raise ValueError("The order of integration must be non-negative") 

819 if len(k) > cnt: 

820 raise ValueError("Too many integration constants") 

821 if np.ndim(lbnd) != 0: 

822 raise ValueError("lbnd must be a scalar.") 

823 if np.ndim(scl) != 0: 

824 raise ValueError("scl must be a scalar.") 

825 if iaxis != axis: 

826 raise ValueError("The axis must be integer") 

827 iaxis = normalize_axis_index(iaxis, c.ndim) 

828 

829 if cnt == 0: 

830 return c 

831 

832 c = np.moveaxis(c, iaxis, 0) 

833 k = list(k) + [0]*(cnt - len(k)) 

834 for i in range(cnt): 

835 n = len(c) 

836 c *= scl 

837 if n == 1 and np.all(c[0] == 0): 

838 c[0] += k[i] 

839 else: 

840 tmp = np.empty((n + 1,) + c.shape[1:], dtype=c.dtype) 

841 tmp[0] = c[0]*0 

842 tmp[1] = c[0]/2 

843 for j in range(1, n): 

844 tmp[j + 1] = c[j]/(2*(j + 1)) 

845 tmp[0] += k[i] - hermval(lbnd, tmp) 

846 c = tmp 

847 c = np.moveaxis(c, 0, iaxis) 

848 return c

851def hermval(x, c, tensor=True): 

852 """ 

853 Evaluate an Hermite series at points x. 

854 

855 If `c` is of length `n + 1`, this function returns the value: 

856 

857 .. math:: p(x) = c_0 * H_0(x) + c_1 * H_1(x) + ... + c_n * H_n(x) 

858 

859 The parameter `x` is converted to an array only if it is a tuple or a 

860 list, otherwise it is treated as a scalar. In either case, either `x` 

861 or its elements must support multiplication and addition both with 

862 themselves and with the elements of `c`. 

863 

864 If `c` is a 1-D array, then `p(x)` will have the same shape as `x`. If 

865 `c` is multidimensional, then the shape of the result depends on the 

866 value of `tensor`. If `tensor` is true the shape will be c.shape[1:] + 

867 x.shape. If `tensor` is false the shape will be c.shape[1:]. Note that 

868 scalars have shape (,). 

869 

870 Trailing zeros in the coefficients will be used in the evaluation, so 

871 they should be avoided if efficiency is a concern. 

872 

873 Parameters 

874 ---------- 

875 x : array_like, compatible object 

876 If `x` is a list or tuple, it is converted to an ndarray, otherwise 

877 it is left unchanged and treated as a scalar. In either case, `x` 

878 or its elements must support addition and multiplication with 

879 with themselves and with the elements of `c`. 

880 c : array_like 

881 Array of coefficients ordered so that the coefficients for terms of 

882 degree n are contained in c[n]. If `c` is multidimensional the 

883 remaining indices enumerate multiple polynomials. In the two 

884 dimensional case the coefficients may be thought of as stored in 

885 the columns of `c`. 

886 tensor : boolean, optional 

887 If True, the shape of the coefficient array is extended with ones 

888 on the right, one for each dimension of `x`. Scalars have dimension 0 

889 for this action. The result is that every column of coefficients in 

890 `c` is evaluated for every element of `x`. If False, `x` is broadcast 

891 over the columns of `c` for the evaluation. This keyword is useful 

892 when `c` is multidimensional. The default value is True. 

893 

894 .. versionadded:: 1.7.0 

895 

896 Returns 

897 ------- 

898 values : ndarray, algebra_like 

899 The shape of the return value is described above. 

900 

901 See Also 

902 -------- 

903 hermval2d, hermgrid2d, hermval3d, hermgrid3d 

904 

905 Notes 

906 ----- 

907 The evaluation uses Clenshaw recursion, aka synthetic division. 

908 

909 Examples 

910 -------- 

911 >>> from numpy.polynomial.hermite import hermval 

912 >>> coef = [1,2,3] 

913 >>> hermval(1, coef) 

914 11.0 

915 >>> hermval([[1,2],[3,4]], coef) 

916 array([[ 11., 51.], 

917 [ 115., 203.]]) 

918 

919 """ 

920 c = np.array(c, ndmin=1, copy=0) 

921 if c.dtype.char in '?bBhHiIlLqQpP': 

922 c = c.astype(np.double) 

923 if isinstance(x, (tuple, list)): 

924 x = np.asarray(x) 

925 if isinstance(x, np.ndarray) and tensor: 

926 c = c.reshape(c.shape + (1,)*x.ndim) 

927 

928 x2 = x*2 

929 if len(c) == 1: 

930 c0 = c[0] 

931 c1 = 0 

932 elif len(c) == 2: 

933 c0 = c[0] 

934 c1 = c[1] 

935 else: 

936 nd = len(c) 

937 c0 = c[-2] 

938 c1 = c[-1] 

939 for i in range(3, len(c) + 1): 

940 tmp = c0 

941 nd = nd - 1 

942 c0 = c[-i] - c1*(2*(nd - 1)) 

943 c1 = tmp + c1*x2 

944 return c0 + c1*x2

1176def hermvander(x, deg): 

1177 """Pseudo-Vandermonde matrix of given degree. 

1178 

1179 Returns the pseudo-Vandermonde matrix of degree `deg` and sample points 

1180 `x`. The pseudo-Vandermonde matrix is defined by 

1181 

1182 .. math:: V[..., i] = H_i(x), 

1183 

1184 where `0 <= i <= deg`. The leading indices of `V` index the elements of 

1185 `x` and the last index is the degree of the Hermite polynomial. 

1186 

1187 If `c` is a 1-D array of coefficients of length `n + 1` and `V` is the 

1188 array ``V = hermvander(x, n)``, then ``np.dot(V, c)`` and 

1189 ``hermval(x, c)`` are the same up to roundoff. This equivalence is 

1190 useful both for least squares fitting and for the evaluation of a large 

1191 number of Hermite series of the same degree and sample points. 

1192 

1193 Parameters 

1194 ---------- 

1195 x : array_like 

1196 Array of points. The dtype is converted to float64 or complex128 

1197 depending on whether any of the elements are complex. If `x` is 

1198 scalar it is converted to a 1-D array. 

1199 deg : int 

1200 Degree of the resulting matrix. 

1201 

1202 Returns 

1203 ------- 

1204 vander : ndarray 

1205 The pseudo-Vandermonde matrix. The shape of the returned matrix is 

1206 ``x.shape + (deg + 1,)``, where The last index is the degree of the 

1207 corresponding Hermite polynomial. The dtype will be the same as 

1208 the converted `x`. 

1209 

1210 Examples 

1211 -------- 

1212 >>> from numpy.polynomial.hermite import hermvander 

1213 >>> x = np.array([-1, 0, 1]) 

1214 >>> hermvander(x, 3) 

1215 array([[ 1., -2., 2., 4.], 

1216 [ 1., 0., -2., -0.], 

1217 [ 1., 2., 2., -4.]]) 

1218 

1219 """ 

1220 ideg = int(deg) 

1221 if ideg != deg: 

1222 raise ValueError("deg must be integer") 

1223 if ideg < 0: 

1224 raise ValueError("deg must be non-negative") 

1225 

1226 x = np.array(x, copy=0, ndmin=1) + 0.0 

1227 dims = (ideg + 1,) + x.shape 

1228 dtyp = x.dtype 

1229 v = np.empty(dims, dtype=dtyp) 

1230 v[0] = x*0 + 1 

1231 if ideg > 0: 

1232 x2 = x*2 

1233 v[1] = x2 

1234 for i in range(2, ideg + 1): 

1235 v[i] = (v[i-1]*x2 - v[i-2]*(2*(i - 1))) 

1236 return np.moveaxis(v, 0, -1)

1239def hermvander2d(x, y, deg): 

1240 """Pseudo-Vandermonde matrix of given degrees. 

1241

1243 points `(x, y)`. The pseudo-Vandermonde matrix is defined by 

1244 

1245 .. math:: V[..., (deg[1] + 1)*i + j] = H_i(x) * H_j(y), 

1246 

1247 where `0 <= i <= deg[0]` and `0 <= j <= deg[1]`. The leading indices of 

1248 `V` index the points `(x, y)` and the last index encodes the degrees of

1250

1256 

1257 and ``np.dot(V, c.flat)`` and ``hermval2d(x, y, c)`` will be the same 

1258 up to roundoff. This equivalence is useful both for least squares 

1259 fitting and for the evaluation of a large number of 2-D Hermite 

1260 series of the same degrees and sample points. 

1261

1267 whether any of the elements are complex. Scalars are converted to 1-D 

1268 arrays. 

1269 deg : list of ints 

1270 List of maximum degrees of the form [x_deg, y_deg]. 

1271 

1272 Returns 

1273 ------- 

1274 vander2d : ndarray 

1275 The shape of the returned matrix is ``x.shape + (order,)``, where 

1276 :math:`order = (deg[0]+1)*(deg([1]+1)`. The dtype will be the same 

1277 as the converted `x` and `y`. 

1278 

1279 See Also

1282 

1283 Notes 

1284 ----- 

1285 

1286 .. versionadded:: 1.7.0 

1287

1306 points `(x, y, z)`. If `l, m, n` are the given degrees in `x, y, z`, 

1307 then The pseudo-Vandermonde matrix is defined by 

1308 

1309 .. math:: V[..., (m+1)(n+1)i + (n+1)j + k] = H_i(x)*H_j(y)*H_k(z), 

1310 

1311 where `0 <= i <= l`, `0 <= j <= m`, and `0 <= j <= n`. The leading 

1312 indices of `V` index the points `(x, y, z)` and the last index encodes 

1313 the degrees of the Hermite polynomials. 

1314 

1315 If ``V = hermvander3d(x, y, z, [xdeg, ydeg, zdeg])``, then the columns 

1316 of `V` correspond to the elements of a 3-D coefficient array `c` of 

1317 shape (xdeg + 1, ydeg + 1, zdeg + 1) in the order 

1318 

1319 .. math:: c_{000}, c_{001}, c_{002},... , c_{010}, c_{011}, c_{012},... 

1320 

1321 and ``np.dot(V, c.flat)`` and ``hermval3d(x, y, z, c)`` will be the 

1322 same up to roundoff. This equivalence is useful both for least squares 

1323 fitting and for the evaluation of a large number of 3-D Hermite 

1324 series of the same degrees and sample points. 

1325 

1326 Parameters 

1327 ---------- 

1328 x, y, z : array_like 

1329 Arrays of point coordinates, all of the same shape. The dtypes will 

1330 be converted to either float64 or complex128 depending on whether 

1331 any of the elements are complex. Scalars are converted to 1-D 

1332 arrays. 

1333 deg : list of ints 

1334 List of maximum degrees of the form [x_deg, y_deg, z_deg]. 

1335 

1336 Returns 

1337 ------- 

1338 vander3d : ndarray 

1339 The shape of the returned matrix is ``x.shape + (order,)``, where 

1340 :math:`order = (deg[0]+1)*(deg([1]+1)*(deg[2]+1)`. The dtype will 

1341 be the same as the converted `x`, `y`, and `z`. 

1342 

1343 See Also 

1344 -------- 

1345 hermvander, hermvander3d. hermval2d, hermval3d 

1346 

1347 Notes 

1348 ----- 

1349 

1350 .. versionadded:: 1.7.0 

1351

1353 ideg = [int(d) for d in deg] 

1354 is_valid = [id == d and id >= 0 for id, d in zip(ideg, deg)] 

1355 if is_valid != [1, 1, 1]: 

1356 raise ValueError("degrees must be non-negative integers")

1358 x, y, z = np.array((x, y, z), copy=0) + 0.0 

1359

1362 vz = hermvander(z, degz)

1367def hermfit(x, y, deg, rcond=None, full=False, w=None): 

1368 """ 

1369 Least squares fit of Hermite series to data. 

1370 

1371 Return the coefficients of a Hermite series of degree `deg` that is the 

1372 least squares fit to the data values `y` given at points `x`. If `y` is 

1373 1-D the returned coefficients will also be 1-D. If `y` is 2-D multiple 

1374 fits are done, one for each column of `y`, and the resulting 

1375 coefficients are stored in the corresponding columns of a 2-D return. 

1376 The fitted polynomial(s) are in the form 

1377 

1378 .. math:: p(x) = c_0 + c_1 * H_1(x) + ... + c_n * H_n(x), 

1379 

1380 where `n` is `deg`. 

1381 

1382 Parameters 

1383 ---------- 

1384 x : array_like, shape (M,) 

1385 x-coordinates of the M sample points ``(x[i], y[i])``. 

1386 y : array_like, shape (M,) or (M, K) 

1387 y-coordinates of the sample points. Several data sets of sample 

1388 points sharing the same x-coordinates can be fitted at once by 

1389 passing in a 2D-array that contains one dataset per column. 

1390 deg : int or 1-D array_like 

1391 Degree(s) of the fitting polynomials. If `deg` is a single integer 

1392 all terms up to and including the `deg`'th term are included in the 

1393 fit. For NumPy versions >= 1.11.0 a list of integers specifying the 

1394 degrees of the terms to include may be used instead. 

1395 rcond : float, optional 

1396 Relative condition number of the fit. Singular values smaller than 

1397 this relative to the largest singular value will be ignored. The 

1398 default value is len(x)*eps, where eps is the relative precision of 

1399 the float type, about 2e-16 in most cases. 

1400 full : bool, optional 

1401 Switch determining nature of return value. When it is False (the 

1402 default) just the coefficients are returned, when True diagnostic 

1403 information from the singular value decomposition is also returned. 

1404 w : array_like, shape (`M`,), optional 

1405 Weights. If not None, the contribution of each point 

1406 ``(x[i],y[i])`` to the fit is weighted by `w[i]`. Ideally the 

1407 weights are chosen so that the errors of the products ``w[i]*y[i]`` 

1408 all have the same variance. The default value is None. 

1409 

1410 Returns 

1411 ------- 

1412 coef : ndarray, shape (M,) or (M, K) 

1413 Hermite coefficients ordered from low to high. If `y` was 2-D, 

1414 the coefficients for the data in column k of `y` are in column 

1415 `k`. 

1416 

1417 [residuals, rank, singular_values, rcond] : list 

1418 These values are only returned if `full` = True 

1419 

1420 resid -- sum of squared residuals of the least squares fit 

1421 rank -- the numerical rank of the scaled Vandermonde matrix 

1422 sv -- singular values of the scaled Vandermonde matrix 

1423 rcond -- value of `rcond`. 

1424 

1425 For more details, see `linalg.lstsq`. 

1426 

1427 Warns 

1428 ----- 

1429 RankWarning 

1430 The rank of the coefficient matrix in the least-squares fit is 

1431 deficient. The warning is only raised if `full` = False. The 

1432 warnings can be turned off by 

1433 

1434 >>> import warnings 

1435 >>> warnings.simplefilter('ignore', RankWarning) 

1436 

1437 See Also 

1438 -------- 

1439 chebfit, legfit, lagfit, polyfit, hermefit 

1440 hermval : Evaluates a Hermite series. 

1441 hermvander : Vandermonde matrix of Hermite series. 

1442 hermweight : Hermite weight function 

1443 linalg.lstsq : Computes a least-squares fit from the matrix. 

1444 scipy.interpolate.UnivariateSpline : Computes spline fits. 

1445 

1446 Notes 

1447 ----- 

1448 The solution is the coefficients of the Hermite series `p` that 

1449 minimizes the sum of the weighted squared errors 

1450 

1451 .. math:: E = \\sum_j w_j^2 * |y_j - p(x_j)|^2, 

1452 

1453 where the :math:`w_j` are the weights. This problem is solved by 

1454 setting up the (typically) overdetermined matrix equation 

1455 

1456 .. math:: V(x) * c = w * y, 

1457 

1458 where `V` is the weighted pseudo Vandermonde matrix of `x`, `c` are the 

1459 coefficients to be solved for, `w` are the weights, `y` are the 

1460 observed values. This equation is then solved using the singular value 

1461 decomposition of `V`. 

1462 

1463 If some of the singular values of `V` are so small that they are 

1464 neglected, then a `RankWarning` will be issued. This means that the 

1465 coefficient values may be poorly determined. Using a lower order fit 

1466 will usually get rid of the warning. The `rcond` parameter can also be 

1467 set to a value smaller than its default, but the resulting fit may be 

1468 spurious and have large contributions from roundoff error. 

1469 

1470 Fits using Hermite series are probably most useful when the data can be 

1471 approximated by ``sqrt(w(x)) * p(x)``, where `w(x)` is the Hermite 

1472 weight. In that case the weight ``sqrt(w(x[i])`` should be used 

1473 together with data values ``y[i]/sqrt(w(x[i])``. The weight function is 

1474 available as `hermweight`. 

1475 

1476 References 

1477 ---------- 

1478 .. [1] Wikipedia, "Curve fitting", 

1479 http://en.wikipedia.org/wiki/Curve_fitting 

1480 

1481 Examples 

1482 -------- 

1483 >>> from numpy.polynomial.hermite import hermfit, hermval 

1484 >>> x = np.linspace(-10, 10) 

1485 >>> err = np.random.randn(len(x))/10 

1486 >>> y = hermval(x, [1, 2, 3]) + err 

1487 >>> hermfit(x, y, 2) 

1488 array([ 0.97902637, 1.99849131, 3.00006 ]) 

1489 

1490 """ 

1491 x = np.asarray(x) + 0.0 

1492 y = np.asarray(y) + 0.0 

1493 deg = np.asarray(deg) 

1494 

1495 # check arguments. 

1496 if deg.ndim > 1 or deg.dtype.kind not in 'iu' or deg.size == 0: 

1497 raise TypeError("deg must be an int or non-empty 1-D array of int") 

1498 if deg.min() < 0: 

1499 raise ValueError("expected deg >= 0") 

1500 if x.ndim != 1: 

1501 raise TypeError("expected 1D vector for x") 

1502 if x.size == 0: 

1503 raise TypeError("expected non-empty vector for x") 

1504 if y.ndim < 1 or y.ndim > 2: 

1505 raise TypeError("expected 1D or 2D array for y") 

1506 if len(x) != len(y): 

1507 raise TypeError("expected x and y to have same length") 

1508 

1509 if deg.ndim == 0: 

1510 lmax = deg 

1511 order = lmax + 1 

1512 van = hermvander(x, lmax) 

1513 else: 

1514 deg = np.sort(deg) 

1515 lmax = deg[-1] 

1516 order = len(deg) 

1517 van = hermvander(x, lmax)[:, deg] 

1518 

1519 # set up the least squares matrices in transposed form 

1520 lhs = van.T 

1521 rhs = y.T 

1522 if w is not None: 

1523 w = np.asarray(w) + 0.0 

1524 if w.ndim != 1: 

1525 raise TypeError("expected 1D vector for w") 

1526 if len(x) != len(w): 

1527 raise TypeError("expected x and w to have same length") 

1528 # apply weights. Don't use inplace operations as they 

1529 # can cause problems with NA. 

1530 lhs = lhs * w 

1531 rhs = rhs * w 

1532 

1533 # set rcond 

1534 if rcond is None: 

1535 rcond = len(x)*np.finfo(x.dtype).eps 

1536 

1537 # Determine the norms of the design matrix columns. 

1538 if issubclass(lhs.dtype.type, np.complexfloating): 

1539 scl = np.sqrt((np.square(lhs.real) + np.square(lhs.imag)).sum(1)) 

1540 else: 

1541 scl = np.sqrt(np.square(lhs).sum(1)) 

1542 scl[scl == 0] = 1 

1543 

1544 # Solve the least squares problem. 

1545 c, resids, rank, s = la.lstsq(lhs.T/scl, rhs.T, rcond) 

1546 c = (c.T/scl).T 

1547 

1548 # Expand c to include non-fitted coefficients which are set to zero 

1549 if deg.ndim > 0: 

1550 if c.ndim == 2: 

1551 cc = np.zeros((lmax+1, c.shape[1]), dtype=c.dtype) 

1552 else: 

1553 cc = np.zeros(lmax+1, dtype=c.dtype) 

1554 cc[deg] = c 

1555 c = cc 

1556 

1557 # warn on rank reduction 

1558 if rank != order and not full: 

1559 msg = "The fit may be poorly conditioned" 

1560 warnings.warn(msg, pu.RankWarning, stacklevel=2) 

1561 

1562 if full: 

1563 return c, [resids, rank, s, rcond] 

1564 else: 

1565 return c

1568def hermcompanion(c): 

1569 """Return the scaled companion matrix of c. 

1570 

1571 The basis polynomials are scaled so that the companion matrix is 

1572 symmetric when `c` is an Hermite basis polynomial. This provides 

1573 better eigenvalue estimates than the unscaled case and for basis 

1574 polynomials the eigenvalues are guaranteed to be real if 

1575 `numpy.linalg.eigvalsh` is used to obtain them. 

1576 

1577 Parameters 

1578 ---------- 

1579 c : array_like 

1580 1-D array of Hermite series coefficients ordered from low to high 

1581 degree. 

1582 

1583 Returns 

1584 ------- 

1585 mat : ndarray 

1586 Scaled companion matrix of dimensions (deg, deg). 

1587 

1588 Notes 

1589 ----- 

1590 

1591 .. versionadded:: 1.7.0 

1592 

1593 """ 

1594 # c is a trimmed copy 

1595 [c] = pu.as_series([c]) 

1596 if len(c) < 2: 

1597 raise ValueError('Series must have maximum degree of at least 1.') 

1598 if len(c) == 2: 

1599 return np.array([[-.5*c[0]/c[1]]]) 

1600 

1601 n = len(c) - 1 

1602 mat = np.zeros((n, n), dtype=c.dtype) 

1603 scl = np.hstack((1., 1./np.sqrt(2.*np.arange(n - 1, 0, -1)))) 

1604 scl = np.multiply.accumulate(scl)[::-1] 

1605 top = mat.reshape(-1)[1::n+1] 

1606 bot = mat.reshape(-1)[n::n+1] 

1607 top[...] = np.sqrt(.5*np.arange(1, n)) 

1608 bot[...] = top 

1609 mat[:, -1] -= scl*c[:-1]/(2.0*c[-1]) 

1610 return mat

1613def hermroots(c): 

1614 """ 

1615 Compute the roots of a Hermite series. 

1616 

1617 Return the roots (a.k.a. "zeros") of the polynomial 

1618 

1619 .. math:: p(x) = \\sum_i c[i] * H_i(x). 

1620 

1621 Parameters 

1622 ---------- 

1623 c : 1-D array_like 

1624 1-D array of coefficients. 

1625 

1626 Returns 

1627 ------- 

1628 out : ndarray 

1629 Array of the roots of the series. If all the roots are real, 

1630 then `out` is also real, otherwise it is complex. 

1631 

1632 See Also 

1633 -------- 

1634 polyroots, legroots, lagroots, chebroots, hermeroots 

1635 

1636 Notes 

1637 ----- 

1638 The root estimates are obtained as the eigenvalues of the companion 

1639 matrix, Roots far from the origin of the complex plane may have large 

1640 errors due to the numerical instability of the series for such 

1641 values. Roots with multiplicity greater than 1 will also show larger 

1642 errors as the value of the series near such points is relatively 

1643 insensitive to errors in the roots. Isolated roots near the origin can 

1644 be improved by a few iterations of Newton's method. 

1645 

1646 The Hermite series basis polynomials aren't powers of `x` so the 

1647 results of this function may seem unintuitive. 

1648 

1649 Examples 

1650 -------- 

1651 >>> from numpy.polynomial.hermite import hermroots, hermfromroots 

1652 >>> coef = hermfromroots([-1, 0, 1]) 

1653 >>> coef 

1654 array([ 0. , 0.25 , 0. , 0.125]) 

1655 >>> hermroots(coef) 

1656 array([ -1.00000000e+00, -1.38777878e-17, 1.00000000e+00]) 

1657 

1658 """ 

1659 # c is a trimmed copy 

1660 [c] = pu.as_series([c]) 

1661 if len(c) <= 1: 

1662 return np.array([], dtype=c.dtype) 

1663 if len(c) == 2: 

1664 return np.array([-.5*c[0]/c[1]]) 

1665 

1666 m = hermcompanion(c) 

1667 r = la.eigvals(m) 

1668 r.sort() 

1669 return r

1672def _normed_hermite_n(x, n): 

1673 """ 

1674 Evaluate a normalized Hermite polynomial. 

1675 

1676 Compute the value of the normalized Hermite polynomial of degree ``n`` 

1677 at the points ``x``. 

1678 

1679 

1680 Parameters 

1681 ---------- 

1682 x : ndarray of double. 

1683 Points at which to evaluate the function 

1684 n : int 

1685 Degree of the normalized Hermite function to be evaluated. 

1686 

1687 Returns 

1688 ------- 

1689 values : ndarray 

1690 The shape of the return value is described above. 

1691 

1692 Notes 

1693 ----- 

1694 .. versionadded:: 1.10.0 

1695 

1696 This function is needed for finding the Gauss points and integration 

1697 weights for high degrees. The values of the standard Hermite functions 

1698 overflow when n >= 207. 

1699 

1700 """ 

1701 if n == 0: 

1702 return np.ones(x.shape)/np.sqrt(np.sqrt(np.pi)) 

1703 

1704 c0 = 0. 

1705 c1 = 1./np.sqrt(np.sqrt(np.pi)) 

1706 nd = float(n) 

1707 for i in range(n - 1): 

1708 tmp = c0 

1709 c0 = -c1*np.sqrt((nd - 1.)/nd) 

1710 c1 = tmp + c1*x*np.sqrt(2./nd) 

1711 nd = nd - 1.0 

1712 return c0 + c1*x*np.sqrt(2)

1715def hermgauss(deg): 

1716 """ 

1717 Gauss-Hermite quadrature. 

1718 

1719 Computes the sample points and weights for Gauss-Hermite quadrature. 

1720 These sample points and weights will correctly integrate polynomials of 

1721 degree :math:`2*deg - 1` or less over the interval :math:`[-\\inf, \\inf]` 

1722 with the weight function :math:`f(x) = \\exp(-x^2)`. 

1723 

1724 Parameters 

1725 ---------- 

1726 deg : int 

1727 Number of sample points and weights. It must be >= 1. 

1728 

1729 Returns 

1730 ------- 

1731 x : ndarray 

1732 1-D ndarray containing the sample points. 

1733 y : ndarray 

1734 1-D ndarray containing the weights. 

1735 

1736 Notes 

1737 ----- 

1738 

1739 .. versionadded:: 1.7.0 

1740 

1741 The results have only been tested up to degree 100, higher degrees may 

1742 be problematic. The weights are determined by using the fact that 

1743 

1744 .. math:: w_k = c / (H'_n(x_k) * H_{n-1}(x_k)) 

1745 

1746 where :math:`c` is a constant independent of :math:`k` and :math:`x_k` 

1747 is the k'th root of :math:`H_n`, and then scaling the results to get 

1748 the right value when integrating 1. 

1749 

1750 """ 

1751 ideg = int(deg) 

1752 if ideg != deg or ideg < 1: 

1753 raise ValueError("deg must be a non-negative integer") 

1754 

1755 # first approximation of roots. We use the fact that the companion 

1756 # matrix is symmetric in this case in order to obtain better zeros. 

1757 c = np.array([0]*deg + [1], dtype=np.float64) 

1758 m = hermcompanion(c) 

1759 x = la.eigvalsh(m) 

1760 

1761 # improve roots by one application of Newton 

1762 dy = _normed_hermite_n(x, ideg) 

1763 df = _normed_hermite_n(x, ideg - 1) * np.sqrt(2*ideg) 

1764 x -= dy/df 

1765 

1766 # compute the weights. We scale the factor to avoid possible numerical 

1767 # overflow. 

1768 fm = _normed_hermite_n(x, ideg - 1) 

1769 fm /= np.abs(fm).max() 

1770 w = 1/(fm * fm) 

1771 

1772 # for Hermite we can also symmetrize 

1773 w = (w + w[::-1])/2 

1774 x = (x - x[::-1])/2 

1775 

1776 # scale w to get the right value 

1777 w *= np.sqrt(np.pi) / w.sum() 

1778 

1779 return x, w
