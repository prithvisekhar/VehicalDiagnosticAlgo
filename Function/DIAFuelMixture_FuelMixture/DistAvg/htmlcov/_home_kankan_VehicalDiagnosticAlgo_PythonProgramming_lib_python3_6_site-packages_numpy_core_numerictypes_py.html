240def obj2sctype(rep, default=None): 

241 """ 

242 Return the scalar dtype or NumPy equivalent of Python type of an object. 

243 

244 Parameters 

245 ---------- 

246 rep : any 

247 The object of which the type is returned. 

248 default : any, optional 

249 If given, this is returned for objects whose types can not be 

250 determined. If not given, None is returned for those objects. 

251 

252 Returns 

253 ------- 

254 dtype : dtype or Python type 

255 The data type of `rep`. 

256 

257 See Also 

258 -------- 

259 sctype2char, issctype, issubsctype, issubdtype, maximum_sctype 

260 

261 Examples 

262 --------

264 <class 'numpy.int32'>

602@set_module('numpy') 

603def find_common_type(array_types, scalar_types): 

604 """ 

605 Determine common type following standard coercion rules. 

606 

607 Parameters 

608 ---------- 

609 array_types : sequence 

610 A list of dtypes or dtype convertible objects representing arrays. 

611 scalar_types : sequence 

612 A list of dtypes or dtype convertible objects representing scalars. 

613 

614 Returns 

615 ------- 

616 datatype : dtype 

617 The common data type, which is the maximum of `array_types` ignoring 

618 `scalar_types`, unless the maximum of `scalar_types` is of a 

619 different kind (`dtype.kind`). If the kind is not understood, then 

620 None is returned. 

621 

622 See Also 

623 -------- 

624 dtype, common_type, can_cast, mintypecode 

625 

626 Examples 

627 -------- 

628 >>> np.find_common_type([], [np.int64, np.float32, complex]) 

629 dtype('complex128') 

630 >>> np.find_common_type([np.int64, np.float32], []) 

631 dtype('float64') 

632 

633 The standard casting rules ensure that a scalar cannot up-cast an 

634 array unless the scalar is of a fundamentally different kind of data 

635 (i.e. under a different hierarchy in the data type hierarchy) then 

636 the array: 

637 

638 >>> np.find_common_type([np.float32], [np.int64, np.float64]) 

639 dtype('float32') 

640 

641 Complex is of a different type, so it up-casts the float in the 

642 `array_types` argument: 

643 

644 >>> np.find_common_type([np.float32], [complex]) 

645 dtype('complex128') 

646 

647 Type specifier strings are convertible to dtypes and can therefore 

648 be used instead of dtypes: 

649 

650 >>> np.find_common_type(['f4', 'f4', 'i4'], ['c8']) 

651 dtype('complex128') 

652 

653 """ 

654 array_types = [dtype(x) for x in array_types] 

655 scalar_types = [dtype(x) for x in scalar_types] 

656 

657 maxa = _can_coerce_all(array_types) 

658 maxsc = _can_coerce_all(scalar_types) 

659 

660 if maxa is None: 

661 return maxsc 

662 

663 if maxsc is None: 

664 return maxa 

665 

666 try: 

667 index_a = _kind_list.index(maxa.kind) 

668 index_sc = _kind_list.index(maxsc.kind) 

669 except ValueError: 

670 return None 

671 

672 if index_sc > index_a: 

673 return _find_common_coerce(maxsc, maxa) 

674 else: 

675 return maxa
