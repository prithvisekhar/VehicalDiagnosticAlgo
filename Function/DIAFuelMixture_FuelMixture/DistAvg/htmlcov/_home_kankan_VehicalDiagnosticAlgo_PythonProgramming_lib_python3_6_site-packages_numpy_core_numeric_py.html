724

727 probability theory, the sum of two independent random variables is

730

732

738 Second one-dimensional input array. 

739 mode : {'full', 'valid', 'same'}, optional 

740 'full': 

741 By default, mode is 'full'. This returns the convolution 

742 at each point of overlap, with an output shape of (N+M-1,). At 

743 the end-points of the convolution, the signals do not overlap 

744 completely, and boundary effects may be seen. 

745 

746 'same': 

747 Mode 'same' returns output of length ``max(M, N)``. Boundary 

748 effects are still visible. 

749 

750 'valid': 

751 Mode 'valid' returns output of length 

752 ``max(M, N) - min(M, N) + 1``. The convolution product is only given 

753 for points where the signals overlap completely. Values outside 

754 the signal boundary have no effect. 

755 

756 Returns 

757 ------- 

758 out : ndarray 

759 Discrete, linear convolution of `a` and `v`. 

760 

761 See Also

764 Transform.

766 polymul : Polynomial multiplication. Same output as convolve, but also 

767 accepts poly1d objects as input. 

768 

769 Notes 

770 ----- 

771 The discrete convolution operation is defined as 

772 

773 .. math:: (a * v)[n] = \\sum_{m = -\\infty}^{\\infty} a[m] v[n - m] 

774 

775 It can be shown that a convolution :math:`x(t) * y(t)` in time/space 

776 is equivalent to the multiplication :math:`X(f) Y(f)` in the Fourier 

777 domain, after appropriate padding (padding is necessary to prevent 

778 circular convolution). Since multiplication is more efficient (faster) 

779 than convolution, the function `scipy.signal.fftconvolve` exploits the 

780 FFT to calculate the convolution of large data-sets. 

781 

782 References 

783 ---------- 

784 .. [1] Wikipedia, "Convolution", 

785 https://en.wikipedia.org/wiki/Convolution 

786 

787 Examples 

788 -------- 

789 Note how the convolution operator flips the second array 

790 before "sliding" the two across one another: 

791 

792 >>> np.convolve([1, 2, 3], [0, 1, 0.5]) 

793 array([0. , 1. , 2.5, 4. , 1.5]) 

794 

795 Only return the middle values of the convolution. 

796 Contains boundary effects, where zeros are taken 

797 into account: 

798 

799 >>> np.convolve([1,2,3],[0,1,0.5], 'same') 

800 array([1. , 2.5, 4. ]) 

801 

802 The two arrays are of the same length, so there 

803 is only one position where they completely overlap: 

804 

805 >>> np.convolve([1,2,3],[0,1,0.5], 'valid') 

806 array([2.5]) 

807 

808 """ 

809 a, v = array(a, copy=False, ndmin=1), array(v, copy=False, ndmin=1)

915 """ 

916 Compute tensor dot product along specified axes. 

917 

918 Given two tensors, `a` and `b`, and an array_like object containing 

919 two array_like objects, ``(a_axes, b_axes)``, sum the products of 

920 `a`'s and `b`'s elements (components) over the axes specified by 

921 ``a_axes`` and ``b_axes``. The third argument can be a single non-negative 

922 integer_like scalar, ``N``; if it is such, then the last ``N`` dimensions 

923 of `a` and the first ``N`` dimensions of `b` are summed over. 

924 

925 Parameters 

926 ---------- 

927 a, b : array_like 

928 Tensors to "dot". 

929 

930 axes : int or (2,) array_like 

931 * integer_like 

932 If an int N, sum over the last N axes of `a` and the first N axes 

933 of `b` in order. The sizes of the corresponding axes must match. 

934 * (2,) array_like 

935 Or, a list of axes to be summed over, first sequence applying to `a`, 

936 second to `b`. Both elements array_like must be of the same length. 

937 

938 Returns 

939 ------- 

940 output : ndarray

942

944 -------- 

945 dot, einsum 

946 

947 Notes 

948 ----- 

949 Three common use cases are: 

950 * ``axes = 0`` : tensor product :math:`a\\otimes b` 

951 * ``axes = 1`` : tensor dot product :math:`a\\cdot b` 

952 * ``axes = 2`` : (default) tensor double contraction :math:`a:b` 

953 

954 When `axes` is integer_like, the sequence for evaluation will be: first 

955 the -Nth axis in `a` and 0th axis in `b`, and the -1th axis in `a` and 

956 Nth axis in `b` last. 

957

960 two sequences of the same length, with the first axis to sum over given 

961 first in both sequences, the second axis second, and so forth.

964 first tensor, followed by the non-contracted axes of the second.

982 >>> d = np.zeros((5,2))

1043 iter(axes) 

1044 except Exception: 

1045 axes_a = list(range(-axes, 0)) 

1046 axes_b = list(range(0, axes))

1051 axes_a = list(axes_a) 

1052 except TypeError:

1098 oldb = [bs[axis] for axis in notin] 

1099

1101 bt = b.transpose(newaxes_b).reshape(newshape_b) 

1102 res = dot(at, bt)

2170 Returns a boolean array where two arrays are element-wise equal within a 

2171 tolerance. 

2172 

2173 The tolerance values are positive, typically very small numbers. The 

2174 relative difference (`rtol` * abs(`b`)) and the absolute difference 

2175 `atol` are added together to compare against the absolute difference 

2176 between `a` and `b`. 

2177 

2178 .. warning:: The default `atol` is not appropriate for comparing numbers 

2179 that are much smaller than one (see Notes). 

2180 

2181 Parameters 

2182 ---------- 

2183 a, b : array_like 

2184 Input arrays to compare. 

2185 rtol : float 

2186 The relative tolerance parameter (see Notes). 

2187 atol : float 

2188 The absolute tolerance parameter (see Notes). 

2189 equal_nan : bool 

2190 Whether to compare NaN's as equal. If True, NaN's in `a` will be 

2191 considered equal to NaN's in `b` in the output array. 

2192 

2193 Returns 

2194 ------- 

2195 y : array_like 

2196 Returns a boolean array of where `a` and `b` are equal within the 

2197 given tolerance. If both `a` and `b` are scalars, returns a single 

2198 boolean value. 

2199 

2200 See Also 

2201 -------- 

2202 allclose 

2203 

2204 Notes 

2205 ----- 

2206 .. versionadded:: 1.7.0 

2207 

2208 For finite values, isclose uses the following equation to test whether 

2209 two floating point values are equivalent. 

2210 

2211 absolute(`a` - `b`) <= (`atol` + `rtol` * absolute(`b`)) 

2212 

2213 Unlike the built-in `math.isclose`, the above equation is not symmetric 

2214 in `a` and `b` -- it assumes `b` is the reference value -- so that 

2215 `isclose(a, b)` might be different from `isclose(b, a)`. Furthermore, 

2216 the default value of atol is not zero, and is used to determine what 

2217 small values should be considered close to zero. The default value is 

2218 appropriate for expected values of order unity: if the expected values 

2219 are significantly smaller than one, it can result in false positives. 

2220 `atol` should be carefully selected for the use case at hand. A zero value 

2221 for `atol` will result in `False` if either `a` or `b` is zero. 

2222 

2223 Examples 

2224 --------

2231 >>> np.isclose([1.0, np.nan], [1.0, np.nan]) 

2232 array([ True, False]) 

2233 >>> np.isclose([1.0, np.nan], [1.0, np.nan], equal_nan=True) 

2234 array([ True, True]) 

2235 >>> np.isclose([1e-8, 1e-7], [0.0, 0.0]) 

2236 array([ True, False]) 

2237 >>> np.isclose([1e-100, 1e-7], [0.0, 0.0], atol=0.0) 

2238 array([False, False]) 

2239 >>> np.isclose([1e-10, 1e-10], [1e-20, 0.0]) 

2240 array([ True, True]) 

2241 >>> np.isclose([1e-10, 1e-10], [1e-20, 0.999999e-10], atol=0.0) 

2242 array([False, True]) 

2243 """

2245 with errstate(invalid='ignore'): 

2246 return less_equal(abs(x-y), atol + rtol * abs(y))

2249 y = asanyarray(b) 

2250 

2251 # Make sure y is an inexact type to avoid bad behavior on abs(MIN_INT). 

2252 # This will cause casting of x later. Also, make sure to allow subclasses 

2253 # (e.g., for numpy.ma). 

2254 dt = multiarray.result_type(y, 1.) 

2255 y = array(y, dtype=dt, copy=False, subok=True) 

2256 

2257 xfin = isfinite(x) 

2258 yfin = isfinite(y) 

2259 if all(xfin) and all(yfin): 

2260 return within_tol(x, y, atol, rtol) 

2261 else: 

2262 finite = xfin & yfin 

2263 cond = zeros_like(finite, subok=True) 

2264 # Because we're using boolean indexing, x & y must be the same shape. 

2265 # Ideally, we'd just do x, y = broadcast_arrays(x, y). It's in 

2266 # lib.stride_tricks, though, so we can't import it here. 

2267 x = x * ones_like(cond) 

2268 y = y * ones_like(cond)

2270 cond[finite] = within_tol(x[finite], y[finite], atol, rtol) 

2271 # Check for equality of infinite values... 

2272 cond[~finite] = (x[~finite] == y[~finite])

2278 cond[both_nan] = both_nan[both_nan]

2280 return cond[()] # Flatten 0d arrays to scalars
