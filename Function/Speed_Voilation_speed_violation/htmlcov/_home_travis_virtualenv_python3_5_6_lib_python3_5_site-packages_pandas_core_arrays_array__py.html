16def array( 

17 data: Sequence[object], 

18 dtype: Optional[Union[str, np.dtype, ExtensionDtype]] = None, 

19 copy: bool = True, 

20) -> ABCExtensionArray: 

21 """ 

22 Create an array. 

23 

24 .. versionadded:: 0.24.0 

25 

26 Parameters 

27 ---------- 

28 data : Sequence of objects 

29 The scalars inside `data` should be instances of the 

30 scalar type for `dtype`. It's expected that `data` 

31 represents a 1-dimensional array of data. 

32 

33 When `data` is an Index or Series, the underlying array 

34 will be extracted from `data`. 

35 

36 dtype : str, np.dtype, or ExtensionDtype, optional 

37 The dtype to use for the array. This may be a NumPy 

38 dtype or an extension type registered with pandas using 

39 :meth:`pandas.api.extensions.register_extension_dtype`. 

40 

41 If not specified, there are two possibilities: 

42 

43 1. When `data` is a :class:`Series`, :class:`Index`, or 

44 :class:`ExtensionArray`, the `dtype` will be taken 

45 from the data. 

46 2. Otherwise, pandas will attempt to infer the `dtype` 

47 from the data. 

48 

49 Note that when `data` is a NumPy array, ``data.dtype`` is 

50 *not* used for inferring the array type. This is because 

51 NumPy cannot represent all the types of data that can be 

52 held in extension arrays. 

53 

54 Currently, pandas will infer an extension dtype for sequences of 

55 

56 ============================== ===================================== 

57 Scalar Type Array Type 

58 ============================== ===================================== 

59 :class:`pandas.Interval` :class:`pandas.arrays.IntervalArray` 

60 :class:`pandas.Period` :class:`pandas.arrays.PeriodArray` 

61 :class:`datetime.datetime` :class:`pandas.arrays.DatetimeArray` 

62 :class:`datetime.timedelta` :class:`pandas.arrays.TimedeltaArray` 

63 ============================== ===================================== 

64 

65 For all other cases, NumPy's usual inference rules will be used. 

66 

67 copy : bool, default True 

68 Whether to copy the data, even if not necessary. Depending 

69 on the type of `data`, creating the new array may require 

70 copying data, even if ``copy=False``. 

71 

72 Returns 

73 ------- 

74 ExtensionArray 

75 The newly created array. 

76 

77 Raises 

78 ------ 

79 ValueError 

80 When `data` is not 1-dimensional. 

81 

82 See Also 

83 -------- 

84 numpy.array : Construct a NumPy array. 

85 Series : Construct a pandas Series. 

86 Index : Construct a pandas Index. 

87 arrays.PandasArray : ExtensionArray wrapping a NumPy array. 

88 Series.array : Extract the array stored within a Series. 

89 

90 Notes 

91 ----- 

92 Omitting the `dtype` argument means pandas will attempt to infer the 

93 best array type from the values in the data. As new array types are 

94 added by pandas and 3rd party libraries, the "best" array type may 

95 change. We recommend specifying `dtype` to ensure that 

96 

97 1. the correct array type for the data is returned 

98 2. the returned array type doesn't change as new extension types 

99 are added by pandas and third-party libraries 

100 

101 Additionally, if the underlying memory representation of the returned 

102 array matters, we recommend specifying the `dtype` as a concrete object 

103 rather than a string alias or allowing it to be inferred. For example, 

104 a future version of pandas or a 3rd-party library may include a 

105 dedicated ExtensionArray for string data. In this event, the following 

106 would no longer return a :class:`arrays.PandasArray` backed by a NumPy 

107 array. 

108 

109 >>> pd.array(['a', 'b'], dtype=str) 

110 <PandasArray> 

111 ['a', 'b'] 

112 Length: 2, dtype: str32 

113 

114 This would instead return the new ExtensionArray dedicated for string 

115 data. If you really need the new array to be backed by a NumPy array, 

116 specify that in the dtype. 

117 

118 >>> pd.array(['a', 'b'], dtype=np.dtype("<U1")) 

119 <PandasArray> 

120 ['a', 'b'] 

121 Length: 2, dtype: str32 

122 

123 Or use the dedicated constructor for the array you're expecting, and 

124 wrap that in a PandasArray 

125 

126 >>> pd.array(np.array(['a', 'b'], dtype='<U1')) 

127 <PandasArray> 

128 ['a', 'b'] 

129 Length: 2, dtype: str32 

130 

131 Finally, Pandas has arrays that mostly overlap with NumPy 

132 

133 * :class:`arrays.DatetimeArray` 

134 * :class:`arrays.TimedeltaArray` 

135 

136 When data with a ``datetime64[ns]`` or ``timedelta64[ns]`` dtype is 

137 passed, pandas will always return a ``DatetimeArray`` or ``TimedeltaArray`` 

138 rather than a ``PandasArray``. This is for symmetry with the case of 

139 timezone-aware data, which NumPy does not natively support. 

140 

141 >>> pd.array(['2015', '2016'], dtype='datetime64[ns]') 

142 <DatetimeArray> 

143 ['2015-01-01 00:00:00', '2016-01-01 00:00:00'] 

144 Length: 2, dtype: datetime64[ns] 

145 

146 >>> pd.array(["1H", "2H"], dtype='timedelta64[ns]') 

147 <TimedeltaArray> 

148 ['01:00:00', '02:00:00'] 

149 Length: 2, dtype: timedelta64[ns] 

150 

151 Examples 

152 -------- 

153 If a dtype is not specified, `data` is passed through to 

154 :meth:`numpy.array`, and a :class:`arrays.PandasArray` is returned. 

155 

156 >>> pd.array([1, 2]) 

157 <PandasArray> 

158 [1, 2] 

159 Length: 2, dtype: int64 

160 

161 Or the NumPy dtype can be specified 

162 

163 >>> pd.array([1, 2], dtype=np.dtype("int32")) 

164 <PandasArray> 

165 [1, 2] 

166 Length: 2, dtype: int32 

167 

168 You can use the string alias for `dtype` 

169 

170 >>> pd.array(['a', 'b', 'a'], dtype='category') 

171 [a, b, a] 

172 Categories (2, object): [a, b] 

173 

174 Or specify the actual dtype 

175 

176 >>> pd.array(['a', 'b', 'a'], 

177 ... dtype=pd.CategoricalDtype(['a', 'b', 'c'], ordered=True)) 

178 [a, b, a] 

179 Categories (3, object): [a < b < c] 

180 

181 Because omitting the `dtype` passes the data through to NumPy, 

182 a mixture of valid integers and NA will return a floating-point 

183 NumPy array. 

184 

185 >>> pd.array([1, 2, np.nan]) 

186 <PandasArray> 

187 [1.0, 2.0, nan] 

188 Length: 3, dtype: float64 

189 

190 To use pandas' nullable :class:`pandas.arrays.IntegerArray`, specify 

191 the dtype: 

192 

193 >>> pd.array([1, 2, np.nan], dtype='Int64') 

194 <IntegerArray> 

195 [1, 2, NaN] 

196 Length: 3, dtype: Int64 

197 

198 Pandas will infer an ExtensionArray for some types of data: 

199 

200 >>> pd.array([pd.Period('2000', freq="D"), pd.Period("2000", freq="D")]) 

201 <PeriodArray> 

202 ['2000-01-01', '2000-01-01'] 

203 Length: 2, dtype: period[D] 

204 

205 `data` must be 1-dimensional. A ValueError is raised when the input 

206 has the wrong dimensionality. 

207 

208 >>> pd.array(1) 

209 Traceback (most recent call last): 

210 ... 

211 ValueError: Cannot pass scalar '1' to 'pandas.array'. 

212 """ 

213 from pandas.core.arrays import ( 

214 period_array, 

215 ExtensionArray, 

216 IntervalArray, 

217 PandasArray, 

218 DatetimeArray, 

219 TimedeltaArray, 

220 ) 

221 from pandas.core.internals.arrays import extract_array 

222 

223 if lib.is_scalar(data): 

224 msg = "Cannot pass scalar '{}' to 'pandas.array'." 

225 raise ValueError(msg.format(data)) 

226 

227 data = extract_array(data, extract_numpy=True) 

228 

229 if dtype is None and isinstance(data, ExtensionArray): 

230 dtype = data.dtype 

231 

232 # this returns None for not-found dtypes. 

233 if isinstance(dtype, str): 

234 dtype = registry.find(dtype) or dtype 

235 

236 if is_extension_array_dtype(dtype): 

237 cls = cast(ExtensionDtype, dtype).construct_array_type() 

238 return cls._from_sequence(data, dtype=dtype, copy=copy) 

239 

240 if dtype is None: 

241 inferred_dtype = lib.infer_dtype(data, skipna=False) 

242 if inferred_dtype == "period": 

243 try: 

244 return period_array(data, copy=copy) 

245 except tslibs.IncompatibleFrequency: 

246 # We may have a mixture of frequencies. 

247 # We choose to return an ndarray, rather than raising. 

248 pass 

249 elif inferred_dtype == "interval": 

250 try: 

251 return IntervalArray(data, copy=copy) 

252 except ValueError: 

253 # We may have a mixture of `closed` here. 

254 # We choose to return an ndarray, rather than raising. 

255 pass 

256 

257 elif inferred_dtype.startswith("datetime"): 

258 # datetime, datetime64 

259 try: 

260 return DatetimeArray._from_sequence(data, copy=copy) 

261 except ValueError: 

262 # Mixture of timezones, fall back to PandasArray 

263 pass 

264 

265 elif inferred_dtype.startswith("timedelta"): 

266 # timedelta, timedelta64 

267 return TimedeltaArray._from_sequence(data, copy=copy) 

268 

269 # TODO(BooleanArray): handle this type 

270 

271 # Pandas overrides NumPy for 

272 # 1. datetime64[ns] 

273 # 2. timedelta64[ns] 

274 # so that a DatetimeArray is returned. 

275 if is_datetime64_ns_dtype(dtype): 

276 return DatetimeArray._from_sequence(data, dtype=dtype, copy=copy) 

277 elif is_timedelta64_ns_dtype(dtype): 

278 return TimedeltaArray._from_sequence(data, dtype=dtype, copy=copy) 

279 

280 result = PandasArray._from_sequence(data, dtype=dtype, copy=copy) 

281 return result
