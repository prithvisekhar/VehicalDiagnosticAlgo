630 """ 

631 result_type(*arrays_and_dtypes) 

632 

633 Returns the type that results from applying the NumPy 

634 type promotion rules to the arguments. 

635 

636 Type promotion in NumPy works similarly to the rules in languages 

637 like C++, with some slight differences. When both scalars and 

638 arrays are used, the array's type takes precedence and the actual value 

639 of the scalar is taken into account. 

640 

641 For example, calculating 3*a, where a is an array of 32-bit floats, 

642 intuitively should result in a 32-bit float output. If the 3 is a 

643 32-bit integer, the NumPy rules indicate it can't convert losslessly 

644 into a 32-bit float, so a 64-bit float should be the result type. 

645 By examining the value of the constant, '3', we see that it fits in 

646 an 8-bit integer, which can be cast losslessly into the 32-bit float. 

647 

648 Parameters 

649 ---------- 

650 arrays_and_dtypes : list of arrays and dtypes 

651 The operands of some operation whose result type is needed. 

652 

653 Returns 

654 ------- 

655 out : dtype 

656 The result type. 

657
